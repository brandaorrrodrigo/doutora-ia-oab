1. FUNDAMENTO E NECESSIDADE

1.1. CONTEXTO JURÍDICO
-----------------------

O sistema DOUTORA IA/OAB opera em área sensível do Direito brasileiro:

[1] Educação jurídica (Lei 9.394/96 - LDB)
[2] Exercício da advocacia (Lei 8.906/94 - Estatuto da OAB)
[3] Responsabilidade civil por informação (CC, Arts. 186, 187, 927)
[4] Proteção de dados pessoais (Lei 13.709/18 - LGPD)
[5] Relações de consumo (Lei 8.078/90 - CDC)

Riscos jurídicos identificados:

  ⚠ Confusão entre ferramenta educacional e consultoria jurídica
  ⚠ Responsabilização por aconselhamento jurídico inadequado
  ⚠ Violação do Estatuto da OAB (exercício ilegal da advocacia)
  ⚠ Não cumprimento de obrigações LGPD
  ⚠ Falta de transparência sobre limitações do sistema


1.2. OBJETIVO DO HARDENING
---------------------------

Implementar controles técnicos e jurídicos para:

  ✓ Delimitar escopo do sistema (educação vs consultoria)
  ✓ Proteger empresa de responsabilização indevida
  ✓ Proteger usuários de uso inadequado
  ✓ Cumprir integralmente a LGPD
  ✓ Assegurar auditabilidade
  ✓ Facilitar defesa em eventuais processos

2. LIMITES EXPLÍCITOS DO SISTEMA
================================================================================

2.1. DEFINIÇÃO DE ESCOPO
-------------------------

ESTE SISTEMA É:

  ✓ Ferramenta educacional de preparação para o Exame da OAB
  ✓ Simulador de questões e situações típicas do exame
  ✓ Recurso de estudo, treinamento e prática
  ✓ Gerador de modelos pedagógicos de peças processuais
  ✓ Sistema de diagnóstico de desempenho em estudos

ESTE SISTEMA NÃO É:

  ✗ Consultoria jurídica
  ✗ Aconselhamento sobre casos reais
  ✗ Substituição de advogado constituído
  ✗ Fonte de orientação jurídica definitiva
  ✗ Garantia de aprovação no Exame da OAB
  ✗ Ferramenta para elaboração de peças em processos reais


2.2. VEDAÇÕES EXPRESSAS
------------------------

É VEDADO o uso deste sistema para:

  [1] Tomar decisões jurídicas em processos reais
  [2] Protocolar peças geradas pelo sistema em tribunais
  [3] Substituir orientação de advogado inscrito na OAB
  [4] Estabelecer relação advogado-cliente
  [5] Oferecer consultoria jurídica remunerada ou gratuita
  [6] Usar conteúdo gerado como fonte única de decisão


2.3. LIMITAÇÕES TÉCNICAS
-------------------------

O sistema possui as seguintes limitações INERENTES:

  [1] Inteligência Artificial:
      - Pode gerar respostas imprecisas ou incorretas
      - Não possui capacidade de julgamento jurídico
      - Não compreende nuances de casos específicos
      - Baseia-se em padrões estatísticos, não em raciocínio jurídico

  [2] Base de conhecimento:
      - Atualização periódica (não em tempo real)
      - Cobertura limitada ao programa do Exame da OAB
      - Não abrange legislação estadual/municipal específica
      - Não considera peculiaridades de casos concretos

  [3] Questões:
      - Baseadas em exames anteriores da OAB
      - Podem não refletir tendências futuras
      - Não substituem estudo completo de doutrina e jurisprudência

  [4] Peças processuais:
      - Modelos genéricos para fins de treinamento
      - Não adaptadas a casos reais
      - Podem conter inadequações para situações específicas
      - Não substituem elaboração profissional

4.2. RETENÇÃO DE LOGS
----------------------

Política de retenção:

  - Logs de ações sensíveis: 5 anos
  - Logs de acesso comum: 6 meses
  - Logs anonimizados: Indefinido (sem dados pessoais)

Anonimização automática:

Função SQL: anonimizar_dados_antigos()

Execução: CRON diário às 3h

  0 3 * * * psql -c "SELECT anonimizar_dados_antigos();"

Ação:
  - Remove usuario_id
  - Remove ip_origem
  - Remove user_agent
  - Remove dados_acessados
  - Marca anonimizado = TRUE
  - Registra data_anonimizacao

Resultado:
  - Log preservado para auditoria estatística
  - Dados pessoais eliminados
  - Compliance com LGPD (minimização de dados)


4.3. AUDITORIA
--------------

Consulta de logs para auditoria:

  -- Logs de um usuário específico
  SELECT tipo_acao, descricao, timestamp
  FROM log_legal
  WHERE usuario_id = :usuario_id
    AND anonimizado = FALSE
  ORDER BY timestamp DESC;

  -- Logs de solicitações LGPD
  SELECT tipo_acao, COUNT(*)
  FROM log_legal
  WHERE tipo_acao LIKE 'solicitacao_lgpd_%'
    AND timestamp >= NOW() - INTERVAL '30 days'
  GROUP BY tipo_acao;

  -- Logs de exclusões de conta
  SELECT COUNT(*), DATE(timestamp)
  FROM log_legal
  WHERE tipo_acao = 'exclusao_conta'
    AND timestamp >= NOW() - INTERVAL '90 days'
  GROUP BY DATE(timestamp);

6.1. DIREITOS DO TITULAR (ART. 18, LGPD)
-----------------------------------------

Tabela: solicitacao_lgpd

Tipos de solicitação:

[1] ACESSO (Art. 18, I e II):
    "Confirmação da existência de tratamento e acesso aos dados"

    Prazo: Imediato (via interface) ou 15 dias (via email)

    Implementação:
      - Interface "Meus Dados" no perfil
      - Exportação em JSON
      - Inclui: cadastro, uso, assinaturas, logs

[2] RETIFICAÇÃO (Art. 18, III):
    "Correção de dados incompletos, inexatos ou desatualizados"

    Prazo: Imediato

    Implementação:
      - Interface "Editar Perfil"
      - Validação de dados
      - Log de alterações

[3] ANONIMIZAÇÃO (Art. 18, IV):
    "Anonimização, bloqueio ou eliminação de dados desnecessários"

    Prazo: 15 dias

    Implementação:
      - Análise manual de necessidade
      - Anonimização de dados históricos
      - Preservação de dados legalmente obrigatórios

[4] PORTABILIDADE (Art. 18, V):
    "Portabilidade dos dados a outro fornecedor"

    Prazo: 15 dias

    Implementação:
      - Exportação completa em JSON
      - Formato estruturado e legível
      - Download via link com expiração

[5] EXCLUSÃO (Art. 18, VI):
    "Eliminação dos dados tratados com consentimento"

    Prazo: Imediato (soft delete) + 30 dias (hard delete)

    Implementação:
      - Soft delete: marca deleted_at
      - Hard delete após 30 dias
      - Preservação de dados legalmente obrigatórios (5 anos fiscal)
      - Anonimização como alternativa

[6] INFORMAÇÃO (Art. 18, VII):
    "Informação sobre compartilhamento"

    Prazo: Imediato (via Política de Privacidade)

    Implementação:
      - Seção "Compartilhamento" na Política
      - Lista clara de terceiros (Stripe, AWS)

[7] REVOGAÇÃO (Art. 18, IX):
    "Revogação do consentimento"

    Prazo: Imediato

    Implementação:
      - Interface "Gerenciar Consentimentos"
      - Desativa funcionalidades dependentes
      - Pode inviabilizar uso do serviço
