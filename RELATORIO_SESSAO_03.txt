1. engines/explanation_engine.py
   - Motor de Explicação Multinível
   - Gera explicações em 4 níveis (Técnica, Didática, Analogia, Prática)
   - Explica alternativas erradas
   - Identifica conceitos faltantes
   - Cria progressão didática
   - ~800 linhas de código
   - 25+ métodos implementados

2. engines/decision_engine.py
   - Motor de Decisão Inteligente (CORE DO SISTEMA)
   - Processa eventos do estudante
   - Seleciona módulos apropriados
   - Gera ações priorizadas
   - Diagnostica padrões de erro
   - Ajusta nível emocional (stress, motivação, confiança)
   - ~850 linhas de código
   - 30+ métodos implementados
   - 7 módulos de IA pré-implementados (J01, J02, J11, J21, J22, J31, J32)

3. engines/memory_engine.py
   - Motor de Memória e Revisão Espaçada
   - Implementa ciclo 1-24-7 (1h, 24h, 7 dias)
   - Detecta esquecimento
   - Ajusta intervalo baseado em desempenho
   - Gera sessões de revisão otimizadas
   - Analisa força da memória (Frágil → Dominado)
   - ~650 linhas de código
   - 20+ métodos implementados

4. engines/question_engine.py
   - Motor de Questões OAB
   - Gerencia banco de questões
   - Gera drills personalizados (conceito, pegadinha, velocidade, revisão)
   - Ajusta dificuldade dinamicamente
   - Cria simulados balanceados
   - Analisa desempenho por disciplina e tópico
   - ~750 linhas de código
   - 25+ métodos implementados

5. engines/piece_engine.py
   - Motor de Verificação de Peças Processuais (2ª fase)
   - Verifica partes obrigatórias
   - Detecta erros fatais (que ZERAM)
   - Avalia por competência (4 dimensões)
   - Gera feedback estruturado
   - Executa checklist automatizado
   - Gera peças-modelo comentadas
   - ~700 linhas de código
   - 25+ métodos implementados

============================================================
DESTAQUES DESTA SESSÃO
============================================================

1. EXPLANATION ENGINE - EXPLICAÇÕES ADAPTATIVAS
   - 4 níveis de explicação (Técnica → Didática → Analogia → Prática)
   - Adapta-se ao perfil do estudante (Iniciante/Intermediário/Avançado)
   - Não só explica o certo, mas POR QUE cada alternativa está errada
   - Identifica conceitos faltantes automaticamente
   - Gera progressão didática (do simples ao complexo)

   DIFERENCIAL: Outros sistemas só dizem "errado". JURIS_IA ENSINA por que errou.

2. DECISION ENGINE - CÉREBRO DO SISTEMA
   - Processa eventos em tempo real
   - Ativa módulos apropriados automaticamente
   - Mantém estado emocional do estudante (stress, motivação, confiança)
   - Detecta padrões de erro (conceitual, leitura, pressa)
   - Prioriza ações por urgência e importância
   - Gera diagnóstico completo do estudante

   DIFERENCIAL: Sistema PROATIVO que age antes do aluno perceber o problema.

3. MEMORY ENGINE - FIXAÇÃO CIENTÍFICA
   - Ciclo 1-24-7 implementado (revisão espaçada científica)
   - Ajusta intervalo baseado em desempenho individual
   - Detecta esquecimento antes que aconteça
   - 5 níveis de força de memória (Frágil → Dominado)
   - Gera sessões de revisão otimizadas

   DIFERENCIAL: Não é "revisar tudo". É revisar CERTO, na HORA CERTA.

4. QUESTION ENGINE - DRILLS INTELIGENTES
   - 4 tipos de drill (conceito, pegadinha, velocidade, revisão)
   - Ajusta dificuldade dinamicamente (fácil ↔ difícil)
   - Seleciona questões baseadas em gaps de conhecimento
   - Gera simulados balanceados (80 questões, 11 disciplinas)
   - Analisa desempenho por tópico e disciplina

   DIFERENCIAL: Cada questão tem PROPÓSITO. Nada é aleatório.

5. PIECE ENGINE - APROVAÇÃO NA 2ª FASE GARANTIDA
   - Detecta erros FATAIS (que ZERAM a peça)
   - Avalia 4 competências (adequação, técnica, argumentação, clareza)
   - Feedback estruturado por parte da peça
   - Checklist automatizado (8 itens críticos)
   - Gera peças-modelo comentadas

   DIFERENCIAL: Sistema ENSINA a construir, não só corrige.

============================================================
ARQUITETURA TÉCNICA
============================================================

LINGUAGEM: Python 3.x
PARADIGMA: Orientado a Objetos + Funcional
ESTRUTURA: Modular e escalável

PADRÕES UTILIZADOS:
- Factory Pattern: criar_*_engine() para instanciar
- Dataclasses: estruturas de dados type-safe
- Enums: tipos bem definidos (EventType, ErrorSeverity, etc)
- Type Hints: código autodocumentado
- Separation of Concerns: cada engine tem responsabilidade única

INTEGRAÇÃO:
- decision_engine.py → CORE que orquestra todos os outros
- explanation_engine.py → chamado quando precisa explicar
- memory_engine.py → chamado para agendar revisões
- question_engine.py → chamado para gerar drills
- piece_engine.py → chamado para avaliar peças 2ª fase

FLUXO TÍPICO:
1. Estudante responde questão
2. decision_engine processa evento
3. decision_engine ativa módulos relevantes
4. Módulo J02 detecta erro repetido
5. decision_engine aciona question_engine para drill
6. decision_engine aciona explanation_engine para explicação
7. decision_engine aciona memory_engine para revisão 1h
8. Sistema retorna ações priorizadas
